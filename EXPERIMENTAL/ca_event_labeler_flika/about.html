ğŸ“Š Quality Assessment Metrics

Comprehensive quality checks ensure high-quality training data:

Class Distribution

  • Pixel counts and percentages per class
  • Background:event ratio (warns if >100:1)
  • Spark:puff:wave ratios
  • Files containing each class

Spatial/Temporal Coverage

  • % frames with annotations
  • % spatial area covered
  • Coverage statistics (mean, std)

Label Consistency

  • Detects isolated single-pixel labels
  • Identifies temporal discontinuities
  • Analyzes size distributions

Annotation Completeness

  • Verifies all images have masks
  • Detects orphan masks
  • Checks file name matching

Inter-Annotator Agreement

  • Cohen's kappa coefficient
  • IoU and Dice scores
  • Pixel accuracy
  • Agreement level classification

ğŸ—‚ï¸ Dataset Creation

Patch Extraction

Extracts overlapping 3D patches from full frames:

  • Default size: 16 frames Ã— 64 Ã— 64 pixels
  • Configurable stride (default: 50% overlap)
  • Only keeps patches with signal

Train/Validation/Test Splitting

  • Training set: 70% (with augmentation)
  • Validation set: 15%
  • Test set: 15%
  • Stratified splitting maintains class balance
  • Reproducible (random seed = 42)

Class Balancing

  • Counts pixels per class in each patch
  • Oversamples minority classes
  • Balances to match majority class
  • Maintains data diversity

Data Augmentation

Creates 2-3 augmented copies per training patch:

  • Geometric: Rotation (90Â°/180Â°/270Â°), flipping
  • Elastic: Smooth deformation with Gaussian displacement
  • Photometric: Brightness, contrast, gamma correction
  • Noise: Gaussian noise addition
  • Temporal: Time warping, frame subsampling
